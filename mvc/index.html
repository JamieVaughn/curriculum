Closure Reminder

var nums = [1,2,3];

for (var i = 0; i < nums.length; i++) {
    // This is the number we're on...
    var num = nums[i]
    // We're creating a DOM element for the number
    var elem = document.createElement('div');
    elem.textContent = num;
    // ... and when we click, alert the value of `num`
    elem.addEventListener('click', (function(numCopy) {
        return function() {
            alert(numCopy);
        };
    })(num));
    document.body.appendChild(elem);
};

The Goal

  • Stable - Bug free and handles errors gracefully
  • Clean code - The code is easily understood by other developers and adheres to standards
  • Scalable - the code can accomodate large numbers of users and/or data
  • Extensible - You can easily add new features without disrupting old ones

Spaghetti Code

  • Things can get messy as you start connecting things together
  • But, applications can't work without connecting different parts together
  • If you connect all parts of your app to every other part, then you can't move anything around anymore
  • You've lost all degrees of freedom and therefore you've lost extensibility, Stability and likely clean code and scalability too
